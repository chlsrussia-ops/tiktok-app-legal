# Инструкции для Claude / Copilot / любого AI-ассистента

> Этот файл — канонический источник правил для AI при работе в этом репо.
> Исходно задумывался как `.claude/instructions.md`, но размещён в корне
> как `CLAUDE.md` (стандартное имя, которое Claude Code подхватывает автоматически).

Правила — часть защиты от ситуации CCE, где 260 файлов в `app/` оказались
proxy-обёртками вида `from src.X import Y`, и правки неделями уходили "в пустоту".

## КРИТИЧНО — проверять ДО любой правки файла

Перед тем как открыть файл на редактирование в `backend/`, `frontend/`, `scripts/`:

1. **Баннер DEPRECATED?**
   Если первая строка / docstring содержит `DEPRECATED` или `@deprecated` —
   **НЕ правь**. Найди актуальную реализацию: грепом по имени функции/класса
   в `src/` (или по пути из `replacement:` в `DEPRECATED_PATHS.yml`).

2. **Это proxy-файл?**
   Если тело файла состоит только из `from ... import ...`
   (± docstring, комментарии, `__all__`), это proxy-обёртка.
   **НЕ правь этот файл.** Иди в настоящий модуль, на который он указывает.
   Автопроверка: `python3 scripts/detect_proxy_wrappers.py <file>`.

3. **Файл попадает в `DEPRECATED_PATHS.yml`?**
   Открой `DEPRECATED_PATHS.yml` в корне репо. Если путь файла начинается
   с любого `path:` из списка — **НЕ пиши туда новый код**, используй
   `replacement:` из той же записи.

4. **Каталог называется `legacy_*`, `old_*`, `deprecated_*`, `_old`, `_legacy`?**
   Это кандидат на удаление. Правки только в режиме "снести", не "допилить".

## Подозрительные паттерны

- Каталог `app/` при наличии `src/` на том же уровне — по умолчанию
  ПОДОЗРИТЕЛЬНЫЙ. Классический след незаконченного рефакторинга.
  Сначала: `grep -rn "def <имя_функции>" .` — где живёт реальный код.
- Два файла с одинаковым именем в разных каталогах (`app/foo.py` и
  `src/foo.py`) — разберись, какой канонический.

## Рабочий процесс

1. Нашёл нужное место → проверил пункты 1–4 выше.
2. Если файл нельзя трогать — сообщи пользователю путь к канонической
   реализации и предложи править её.
3. Нашёл ненужную proxy-обёртку — предложи удалить (не удаляй сам без
   явного подтверждения — см. правила пользователя в MEMORY.md).
4. Создаёшь новый файл — никогда не клади его в deprecated-путь.

## Полезные команды

```bash
# Все proxy-обёртки в backend
python3 scripts/detect_proxy_wrappers.py --root backend

# Проставить баннеры DEPRECATED где требуется
python3 scripts/add_legacy_banner.py

# Убедиться, что баннеры на месте (CI-style)
python3 scripts/add_legacy_banner.py --check

# Найти канонический модуль для файла
grep -rn "from $(basename FILE .py)" src/ backend/app/
```

## Если сомневаешься

Спроси пользователя явно: "Файл X похож на proxy/legacy, уточни, где
каноническая реализация?". Лучше переспросить, чем править мёртвый код.
